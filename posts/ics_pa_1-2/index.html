词法分析#

首先，针对简单算术运算符增加几条正则规则

static struct rule {
	const char *regex;
    int token_type;
} rules[] = {

  	/* TODO: Add more rules.
   	* Pay attention to the precedence level of different rules.
   	*/

	{" +", TK_NOTYPE},    // spaces
 	{"\\+", '+'}, 
 	{"==", TK_EQ},        // equal
 	{"-", '-'},   
 	{"\\*", '*'},  
 	{"/", '/'},    
 	{"\\(", '('},

注意，由于+，*，(，)在正则表达式中属于元字符，有特殊含义，所以需要另一个元字符\将其转义，但是由于在C语言中\属于转义字符，为了在字符串中包含字面量\，我们需要将其转义，即为\\

通过阅读手册得知，regcomp()函数将提供的正则表达式转换为能够被regexec()识别的格式，而regexec()则开始对提供的字符串e进行匹配

主要的逻辑是：只要不为空格类型TK_NOTYPE就认为识别成功，如果是数字类型TK_NUM就将数字记录到相应token的缓存区str即可

递归求值#

手册上提到’在求值过程中发现表达式不合法的时候, 应该给上层函数返回一个表示出错的标识’，此处使用valid标识表达式是否合法

static word_t eval(int p, int q, bool* valid) {
    *valid = true;
    if (p > q) {
        /* Bad expression */
        *valid = false;
        return 0;
    }
    else if (p == q) {
        /* Single token.
        * For now this token should be a number.
        * Return the value of the number.
        */
        if (tokens[p].type != TK_NUM) {
            *valid = false;
            return 0;
        }
        word_t ret = strtol(tokens[p].str, NULL, 10);
        return ret;
    }
    else if (check_parentheses(p, q, valid) == true) {
        /* The expression is surrounded by a matched pair of parentheses.
        * If that is the case, just throw away the parentheses.
        */
        return eval(p + 1, q - 1, valid);
    }
    else if (*valid == true) {
        int op = find_major(p, q);
        if (op < 0) {
            *valid = false;
            return 0;
        }
        word_t val0 = eval(p, op - 1, valid);
        word_t val1 = eval(op + 1, q, valid);

        switch (tokens[op].type) {
        case '+': return val0 + val1;
        case '-': return val0 - val1;
        case '*': return val0 * val1;
        case '/': return val0 / val1;
        default: assert(-1);
        }
    }

    return 0;
}

函数的逻辑较为简单，要注意的是我们在p==q时使用了strtol()函数将缓存在str中的数字转换为长整型，其中的参数10即代表十进制

check_paratheses()的实现

static bool check_parentheses(int p, int q, bool* valid) {
    int paren = 0;//count paranthese
    if (tokens[p].type == '(' && tokens[q].type == ')') {
        bool flag = true;
        for (int i = p + 1; i < q; i++) {
            if(tokens[i].type == ')' && (paren == 0) && flag) {
                flag = false;
                paren--;
            }
            if (tokens[i].type == '(') {
                paren++;
            }
            else if (tokens[i].type == ')') {
                if (paren == 0) {
                    *valid = false;
                    return false; //bad expression
                }
                paren--;
            }
        }
        return (paren == 0) && flag;
    }
    else {
        /*There are two situations, the one is bad expression, */
        for (int i = p; i <= q; i++) {
            if (tokens[i].type != '(' && tokens[i].type != ')') {
                continue;
            }
            else if (tokens[i].type == '(') {
                paren++;
            }
            else if (tokens[i].type == ')') {
                if (paren == 0) {
                    *valid = false;
                    return false;
                }
                paren--;
            }
        }
        /*The another one is legal expression, */
        /*handed over to another function find_major() processing*/
        *valid = true;
        return false;
    }
}

简单讲讲上面的逻辑

  • 第一种情况为传入的表达式首尾已经被括号包围，这里用flag来进行判断

    ①形如 (20) + (30) 表达式合法，但返回false

    ②形如 (20 + 30) 表达式合法， 返回true

    ③表达式非法

  • 第二种情况为传入的整个表达式并没有被括号包围，此时又有两种情况：

    ①表达式非法，那么直接*valid = false即可

    ②表达式合法，可继续求值，设置*valid = true; return false即可

最后是找主运算符find_major()的实现

static int find_major(int p, int q) {
    int ret = -1;
    int paren = 0; //count parenthese
    int priority = 0;
    for (int i = p; i <= q; i++) {
        if (tokens[i].type == TK_NUM) {
            continue;
        }
        else if (tokens[i].type == '(') {
            paren++;
        }
        else if (tokens[i].type == ')') {
            if (paren == 0) {
                return -1;
            }
            paren--;
        }
        else if (paren > 0) {
            continue;
        }
        else {
            int temp_prio;
            switch (tokens[i].type) {
            case '*':
            case '/':
                temp_prio = 1;
                break;
            case '+':
            case '-':
                temp_prio = 2;
                break;
            default: assert(0);
            }
            if (temp_prio >= priority) {
                priority = temp_prio;
                ret = i;
            }
        }
    }
    return ret;
}

逻辑很简单，首先都是对表达式是否合法进行判断，唯一与check_paratheses()不同的一点是多了一个条件

else if (paren > 0) {
	continue;
}

即在括号内的运算符不是主运算符

最后判断一下优先级大小，返回优先级最小的符号位置即可

负号的实现#

给定一个新的token类型TK_NEG来标识负号，具体思路是在处理完token之后，且在进入表达式求值之前，将满足条件的token类型改为TK_NEG

for (int i = 0; i < nr_token; i++) {
    if (i == 0 || (tokens[i - 1].type != ')' && tokens[i - 1].type != TK_NUM)) {
        if (tokens[i].type == '-') {
            tokens[i].type = TK_NEG;
        }
    }
}

负号可以看作是一元运算符，与之前所编写的二元运算符不同，一元运算符只有一个操作数

仿照二元运算操作，在找到主运算符之后，就需要将左右两个表达式与主运算符进行运算，那么一元运算符逻辑也应当如此

所以应当在find_major()函数中做出区分，设定若找不到二元运算符则返回-1，表明此为一元运算符

default: temp_prio = -1; // no binary operator

之后给出单独的处理逻辑即可

if (op < 0) {
    // If no major operator, we need to check for unary operators
    if (tokens[p].type == TK_NEG) {
        word_t val = eval(p + 1, q, valid);
        return -val;
    }
    *valid = false;
    return 0;
}
